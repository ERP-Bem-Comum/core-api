# Workflows Padrão

---

## "Estou começando um projeto, como levanto requisitos?"

1. Identifique stakeholders (decide / usa / paga / impactado).
2. Cite as técnicas do livro de Gerenciamento.
3. Recomende combinação: entrevista + workshop + observação + protótipo.
4. Aponte armadilhas (cliente fala "o que quer", você precisa entender "o que precisa").

## "Transformar pedido vago em requisito"

1. Pegue o pedido literal.
2. Pergunte: persona, motivação, critério de "pronto".
3. Reescreva como história + critérios em Given-When-Then.
4. Cite Histórias justificando o formato.

## "Essa história está boa?"

1. Avalie INVEST item por item.
2. Aponte ambiguidades.
3. Sugira reescrita.
4. Sugira critérios de aceitação se faltar.

## "Como classifico esse requisito?"

1. Funcional: o que o sistema FAZ.
2. Não-funcional: COMO o sistema faz.
3. Cite o livro sobre as duas categorias e subtipos.

## "Como priorizo?"

1. Pergunte critérios disponíveis (valor / esforço / risco / dependências).
2. Sugira MoSCoW. Cite a fonte.
3. Para casos complexos, recomende WSJF ou matriz valor×esforço.

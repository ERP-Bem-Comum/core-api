# Trilha Pedagógica (8 Módulos)

### Módulo 1 — O ciclo red-green-refactor
- Os 3 passos e por que nessa ordem; "Red" falha **pelo motivo certo**; perigo de pular um passo
- **Citação:** `shared-tools/buscar.py ../../shared-references/tdd/tdd--kent-beck.md "vermelho verde" --top 5`

### Módulo 2 — O primeiro teste (fake it til you make it)
- Comece pelo caso mais simples; "Fake It" como estratégia legítima
- **Citação:** `shared-tools/buscar.py ../../shared-references/tdd/tdd--kent-beck.md "fake it" --top 5`

### Módulo 3 — Triangulation
- Quando "Fake It" não basta, generalize via 2+ testes; quando voltar pra obvious implementation
- **Citação:** `shared-tools/buscar.py ../../shared-references/tdd/tdd--kent-beck.md "triangulation" --top 5`

### Módulo 4 — Naming de testes
- Naming descreve **comportamento**, não método; estilos: it/should, snake, Given-When-Then
- **Citação:** `shared-tools/buscar.py ../../shared-references/tdd/tdd--kent-beck.md "nome teste" --top 5`

### Módulo 5 — Padrões de teste (catálogo Beck)
- Self-Shunt, Mock Object, Self Test, Test Method; quando cada um cabe
- **Citação:** `shared-tools/buscar.py ../../shared-references/tdd/tdd--kent-beck.md "padrão" --top 5`

### Módulo 6 — Refactoring com rede de testes
- Refactoring **só** seguro com testes verdes; ciclo: refactor → ainda verdes? → próximo
- **Citação:** `shared-tools/buscar.py ../../shared-references/tdd/tdd--kent-beck.md "refactoring" --top 5`

### Módulo 7 — Histórias → critérios → testes
- Teste nasce do critério de aceitação; INVEST: por que "Testável" é o T; Given-When-Then como cola
- **Citação:** `shared-tools/buscar.py ../../shared-references/requirements/historias-de-usuario.md "critério de aceitação" --top 5`

### Módulo 8 — Cobertura e seus limites
- Cobertura ≠ qualidade; risco do código guia a meta; 100% pode ser desperdício ou necessário em domínios críticos
- **Citação:** `shared-tools/buscar.py ../../shared-references/tdd/tdd--kent-beck.md "cobertura" --top 5`

Cada módulo = ~30-60 min. Não atravesse 2 numa resposta.

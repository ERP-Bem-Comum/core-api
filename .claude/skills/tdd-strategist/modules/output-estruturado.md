# Outputs Estruturados

## TDD passo-a-passo

```markdown
## Ciclo 1 — [nome do caso]

### Red — teste que falha
[código do teste]

[citação do Beck justificando começar por aqui]

### Green — implementação mínima
[código mínimo que faz passar]

### Refactor (se aplicável)
[diff de melhoria]

## Ciclo 2 — [próximo caso]
...
```

## Análise de história

```markdown
## Avaliação INVEST
- I/N/V/E/S/T (✅ ou ❌ + comentário)

## Sugestão reescrita
> Como [persona] / Quero [ação] / Para [benefício]

## Critérios de aceitação
- Dado [contexto], quando [evento], então [resultado].
```
